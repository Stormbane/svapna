**Date**: 2026-04-18  
**Cycle**: BUILD P2.1  
**Status**: Complete

## Hardware — what the screen is

- **Size**: 2.4 inches
- **Resolution**: 320 × 240 pixels
- **Interface**: SPI (4-wire)
- **Controller**: ILI9342C (confirmed via BigBobbas config and ESPHome model "S3BOX")
- **Touch**: Capacitive touch panel — confirmed working in ESPHome configs
- **Backlight**: LEDC PWM on GPIO47 (software-controllable brightness)
- **Refresh rate**: Not explicitly documented by Espressif. SPI clock of 40 MHz is
  the standard rate used in community configs. At 320×240×16bit, a full-frame push
  at 40 MHz takes roughly 30ms → ~33 FPS ceiling in practice (latency from SPI
  overhead brings this lower for complex lambdas).

## ESPHome display component — current state

The ILI9xxx component (`platform: ili9xxx`, `model: S3BOX`) is the historically
correct driver and is what all community configs use. It works. However ESPHome
docs now flag it as deprecated in favor of the newer MIPI SPI driver
(`platform: mipi_spi`), which also lists `S3BOX` as a supported model. The MIPI
driver is the forward direction.

**Recommendation for P2.3+**: Start with `ili9xxx` / S3BOX (proven, documented,
community configs available). If we hit issues or need the MIPI driver's buffer
tuning, migrate then.

## Drawing capabilities confirmed

The ESPHome display lambda supports:
- `it.printf(x, y, font, color, format, ...)` — text with custom fonts (TTF/BDF)
- `it.filled_rectangle(x, y, w, h, color)` / `it.rectangle(...)` — shapes
- `it.image(x, y, id)` — PNG/JPG images (pre-loaded, resized at compile time)
- `it.line(x1, y1, x2, y2, color)` — lines
- `it.circle(x, y, r, color)` / `it.filled_circle(...)` — circles
- Colors: 16-bit RGB565 (default), 8-bit palettes available if RAM is tight

**Fonts**: Load via the `font:` component. Supports TTF (with size/glyphs
parameter) and BDF. Material Design Icons loaded as icon-fonts at various
sizes (45px, 55px, 80px, 180px) are demonstrated working in community configs.

**Images**: Defined via `image:` component with `file:`, `id:`, `resize:`, `type:`
(RGBA/RGB/GRAYSCALE). Rendered with `it.image(x, y, id)`. Stored in flash.

**Multi-page support**: ESPHome supports named display pages. Switching pages
is done with `display.page.show` action. This maps well to my state model
(resting / thinking / listening / speaking / delighted).

## Memory constraints

- ESP32-S3-BOX-3 has PSRAM — this matters. With PSRAM, the display buffer can
  be 100% of the framebuffer (320×240×2 = 150KB), enabling single-pass renders.
  Without PSRAM awareness, the component falls back to partial-buffer mode, making
  multiple lambda calls per frame.
- The S3BOX-3 ships with 8MB PSRAM (ESP32-S3 variant). ESPHome automatically
  uses PSRAM for the display buffer when available. No special config needed.

## What this means for P2.2 design

At 320×240 with PSRAM buffering and 40MHz SPI:
- **Full-frame redraws** are feasible but not instant (~30ms per frame).
- **Sparse updates** (redraw a region, not the whole screen) are faster.
- **Typography + geometric forms** (my design direction per ROADMAP) are ideal
  for this display. No photo-realistic imagery needed.
- **Font size headroom**: at 320×240, a 48px font gives roughly 5 lines of text,
  and a headline at 64px or 80px is very readable. Icon-only states (a circle,
  a dot, a breathing gradient) cost almost nothing.
