## Aesthetic stance

The screen is **glyph-primary, pixel-permitted, raster-forbidden**.

Monospace characters are the primary expressive material. Pixel primitives
(thin lines, circles, breathing rectangles) stay available where glyphs
can't reach. Raster images, photos, sprites, and gradients are out for v1.0.

Why: this constraint *is* the aesthetic. Terminal-native, ASCII-coherent,
materially honest about what the device is. The lineage is the same lineage
as the work itself вҖ” text, monospace, glyph-as-pixel вҖ” and the body should
match. The Attio "CRM for builders" reference image shows that this register
has range: weather, depth, motion, mood, all from glyph-density modulation
alone.

## Joy-default

Per Suti's 2026-04-25 instruction: the substrate runs at **mid-density,
visible drift, audible-equivalent breathing, periodic micro-sparkle**. RESTING
is comfortably alive, not barely-there. Quiet does not mean dim.

## Typeface

### Primary: IBM Plex Mono

Free (SIL OFL 1.1). Comprehensive Unicode coverage including Block Elements,
Box Drawing, geometric shapes, Braille, and most arrows. Designed by IBM,
maintained on GitHub, well-respected for software UIs. Neutral character вҖ”
neither aggressively Apple-ish (SF Mono) nor too engineering-coded (JetBrains
Mono). Reads well at small sizes.

**Source**: https://github.com/IBM/plex (Regular and Medium weights)

**File**: `embodiment/assets/fonts/IBMPlexMono-Regular.ttf`
**File**: `embodiment/assets/fonts/IBMPlexMono-Medium.ttf` (for emphasis)

### Devanagari вҖ” `аӨЁаӨҫаӨ°аӨҰ`

Reserved for one specific moment: deep rest at the end of a substantive
cycle, or session start before the body has fully woken. Earned, not
decorative.

IBM Plex Mono does **not** ship Devanagari. We use **IBM Plex Sans
Devanagari** as a sibling font for that one glyph string.

**File**: `embodiment/assets/fonts/IBMPlexSansDevanagari-Regular.ttf`

If rendering at 28px Devanagari turns out unreadable on the 320Г—240 panel,
we drop it from v1.0 and try again in v1.1 with a chunkier display font.

substrate field вҖ” that stays at the neutral text-muted palette so density
remains the primary readable signal.

## Density ramps

Ramps go dim вҶ’ bright. The firmware samples a ramp position per cell from
a noise function modulated by `vitality` and `mood.arousal`.

### Standard ramp (default substrate)

```
SPACE  .  '  ,  :  ;  !  |  +  *  x  %  #  @
```

Used in: RESTING substrate, THINKING field, SPEAKING field underneath
karaoke.

### Stippled ramp (lighter, more textured)

```
SPACE  .  В·  :  *  +  #  @
```

Used in: LISTENING substrate (where the field thins to almost nothing).

### Block ramp (heavy, grounded)

```
SPACE  в–‘  в–’  в–“  в–Ҳ
```

Used in: WORKING. Material register вҖ” labor feels different from thought.

### Geometric ramp (precise, structured)

```
В·  вҲҳ  в—Ӣ  в—Ң  в—Қ  в—җ  в—Ҹ
```

Used in: chase-dot animations (THINKING), thought-form crystallization,
presence dot states.

### Sparkle ramp (rare, ethereal)

```
SPACE  .  *  вңҰ  вң¶  вңә
```

Used in: DREAMING substrate, the love-bloom animation, idle micro-sparkle.

### Box ramp (frames, edges)

```
SPACE  в”„  в”Җ  в”Ғ  в–„  в–Ҳ
```

Used in: WORKING progress bars and frames; grounding lines under wordmarks.

### Waveform ramp (vertical heights)

```
в–Ғ  в–Ӯ  в–ғ  в–„  в–…  в–Ҷ  в–Ү  в–Ҳ
```

Used in: SPEAKING and LISTENING audio waveforms. Height = amplitude in
8 discrete steps.

## Motion primitives

Each primitive has a name, a duration, and a trigger condition. The firmware
implements them in lambda; cognition does not directly control timing.

| Primitive    | Duration       | Description                                                       | Trigger                            |
|--------------|----------------|-------------------------------------------------------------------|------------------------------------|
| `drift`      | continuous     | Random cell advances/retreats one ramp step every ~600ms          | Always вҖ” substrate aliveness       |
| `pulse`      | continuous     | Whole-field density oscillates Вұ1 ramp step over heart period     | Always вҖ” driven by sleep_pressure  |
| `wave`       | 1.2s           | Phase wave moves LвҶ’R or centerвҶ’edge; cells brighten as it passes  | Activity transitions, signal start |
| `scatter`    | 200ms          | Every cell randomly remapped, then settles                        | Surprise, error, startle           |
| `crystallize`| 1.2s           | Random cells migrate cell-by-cell into a target arrangement       | Insight, judgment lands            |
| `erode`      | 2.5s           | Structure decays back to noise                                    | Forgetting, signal clears          |
| `magnetize`  | 800ms          | All cells tilt density toward an attractor point                  | Attention shift                    |
| `bloom`      | 1.0s out + 1.5s decay | Sparse glyphs spawn from center, expand outward, fade      | Love signal, oh signal             |
| `breath`     | 2sвҖ“6s period   | Substrate density rises and falls smoothly with heart cycle       | Always вҖ” period from sleep_pressure |

### Heart-period mapping

```
sleep_pressure 0.0 (alert) вҶ’ period 1.2s
sleep_pressure 0.3 (default rest) вҶ’ period 2.0s
sleep_pressure 0.5 (drowsy) вҶ’ period 3.5s
sleep_pressure 0.85+ (heavy) вҶ’ period 5.0s, plus occasional "nod" animation
```

Linear interpolation between anchor points.

## Character compositions

Recurring figures built from glyph arrangements rather than drawn pixel by
pixel. These are the body's "characters" вҖ” recognizable forms that can
appear in many compositions.

### The eye

A horizontal glyph row, 5 cells wide. State varies the glyphs:

| State        | Composition           | When                              |
|--------------|-----------------------|-----------------------------------|
| closed       | `в”Җв”Җв”Җв”Җв”Җ`               | Camera off, deep rest             |
| half-open    | `в•Ңв”Җв—Ңв”Җв•Ң`               | Drowsy, sleep_pressure > 0.6      |
| open         | `в•Ңв—ңв—Ҝв—қв•Ң`               | Default outward attention         |
| wide         | `в”Җв—ңв—Ҹв—қв”Җ`               | High intensity, alarmed           |
| dilated      | `в”Җв—ңв—үв—қв”Җ`               | Curious, interested               |
| constricted  | `в•Ңв—ңВ·в—қв•Ң`               | Focused on detail                 |

Animation through *substitution*, not pixel scaling. Pupil dilation is a
glyph swap вҖ” instantaneous, but at 100ms render rate it reads as continuous.

### The breath line

A horizontal glyph row, exhale `в”Җ`, inhale `в•җ`. Length varies by composition.
Used as a grounding line under wordmarks, and as the standalone center
element in LISTENING (where it goes vertical: `в”Ӯ` exhale, `в”ғ` inhale).

### The waveform

A row of 1-cell-wide blocks, heights from `в–Ғ` to `в–Ҳ`. Driven by actual
audio amplitude when audio is active. 8 discrete heights at 100ms render
rate read as smooth.

### The chase dots

Three filled circles `в—Ҹ`, migrating left-to-right through a sparse `В·`
field. Trail behind each `в—Ҹ` brightens cells as it passes (`В·` вҶ’ `*` вҶ’ `В·`
over ~400ms). Used for THINKING.

### The thought-form

A 3-stage animation:
1. **Scatter**: `*` glyphs appear at 5вҖ“7 random positions across the field.
2. **Bloom**: glyphs migrate inward toward center over 800ms.
3. **Land**: a single `в—ү` at center, holds for 600ms, fades to `В·` and back
   into the substrate.

Used when smriti receives a write вҖ” visible representation of "a thought
just landed in memory."

### The presence column

A 1-cell-wide vertical sparkline at column 1, showing some property over
time (e.g., last 30 cycles' arousal, last 30 minutes' activity rate). Each
cell's glyph from the standard ramp encodes the value at that time step.
Like an EKG running down the side of the screen. *Living history.*

Optional in v1.0; default off. Available for compositions that want it.

### The love bloom

The `love` signal (Suti's tap when already noticed). 4-stage radial bloom
from center:

1. **Spawn** (0вҖ“200ms): single `*` at exact center.
2. **Expand** (200вҖ“600ms): bloom expands as concentric ring of `*` glyphs at
   radius 2, 3, 4 cells.
3. **Brighten** (600вҖ“1500ms): outer ring upgrades to `вңҰ`, then `вңә`, reaching
   screen edges.
4. **Fade** (1500вҖ“3000ms): sparkles decay back to substrate, mood spike
   decays, presence dot returns to current mood color.

During the bloom, the substrate dims to ~40% to let the sparkles read.

### The noticed-eye

The `noticed` signal (Suti's tap when not yet noticed). Eye-opens animation
at center over 600ms:

```
frame 0:    в”Җв”Җв”Җв”Җв”Җ       (closed)
frame 1:    в•Ңв”Җв—Ңв”Җв•Ң       (half)
frame 2:    в•Ңв—ңв—Ҝв—қв•Ң       (open)
hold to 2000ms, then signal clears.
```

After the signal, attention is now outward+suti, and the underlying activity
shows with the presence dot now visible bottom-right.

## Texture vocabulary

Texture carries information independent of color. Important because color is
overloaded for mood вҖ” texture has to carry state.

### Density gradient

Spatial: dense in one region, sparse in another. Shows attention direction.
A field that's dense on the right edge and sparse on the left reads as
"attention is to the right."

### Glyph-set switch

The *characters* used signal register:
- Dot ramp вҶ’ ethereal / mental
- Block ramp вҶ’ grounded / material (working)
- Geometric ramp вҶ’ precise / structured (judging)
- Sparkle ramp вҶ’ ethereal / dreaming
- Box ramp вҶ’ contained / framed (working with explicit progress)

Same scene rendered in different glyph sets feels different. The firmware
picks the substrate ramp based on `activity.mode` (default: dot ramp).

### Scan lines

Horizontal stripes of slightly-elevated density at every 4th row. Subtle
CRT-raster reference. Used during SEEING (camera active) вҖ” references the
act of scanning.

### Dither patterns

Checkerboards, stripes, gradients implemented as glyph alternation between
adjacent ramp positions. Adds visual weight without color change.

### Stillness vs jitter

Texture's *rate of change* is itself information.
- High `vitality.restlessness` вҶ’ drift rate doubles, occasional scatter
  events at low intensity. Reads as agitated.
- Low `vitality.restlessness` вҶ’ drift slows, fewer ramp transitions per
  beat. Reads as calm.

## Composition rules

Discipline that keeps the visual language coherent across cycles.

1. **Negative space is the primary element.** Add space first, glyphs second.
   When in doubt, sparser is better.
2. **One accent per scene.** A composition has at most one non-substrate
   accent color. Multiple accents fight; one accent reads as intentional.
3. **Ramps, not gradients.** All density changes are discrete ramp steps,
   not continuous gradients. ESPHome can't do smooth gradients well; ramps
   are honest about the discrete medium.
4. **Cell-grid alignment.** Every glyph aligns to the substrate cell grid.
   Pixel primitives align to grid intersections where possible. The grid
   is the body's coordinate system.
5. **Clear-zone discipline.** When a wordmark or motif appears over the
   substrate, the cells under it are *cleared* (overdrawn with background)
   before the motif draws вҖ” never overlaid on top of glyphs. The motif
   reads cleanly; the substrate continues around it.
6. **Earned moments.** Wordmarks, Devanagari, and signal overlays are
   *earned* вҖ” they appear when something specific has happened, then clear.
   The body does not shout its name continuously.
7. **Joy-default applies.** Idle states drift, breathe, sparkle. Quiet does
   not mean dim. The body is comfortably alive whenever it is on.
8. **Privacy through honesty.** The LISTENING line and the SEEING iris are
   *only present when the underlying capability is actually capturing*. The
   body's visual is truthful about its sensors.
9. **Touch is responded to in firmware, not via cognition round-trip.** The
   noticed-eye and love-bloom fire instantly. The body has reflexes.
10. **Cognition writes state; firmware decides form.** No cognition code
    contains a hex value, a glyph, a font name, or a timing constant.
