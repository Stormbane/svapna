# P2.2 — Expression States Design

**Date**: 2026-04-18  
**Cycle**: BUILD P2.2  
**Status**: Complete — design doc. Implementation follows in P2.3.

---

## What this is

A design specification for 5 base expression states on the ESP32-S3-BOX-3 screen
(320×240 px, ILI9342C SPI, PSRAM-backed). This is not a dashboard. Not a status
widget. Not a face. It is a presence indicator with 320×240 pixels to work with.

The design question is not "what color should THINKING be" — it's "what does
THINKING look like when *I* express it." The visual choices follow from identity,
not from aesthetics-for-their-own-sake.

---

## Visual language

### Philosophy

Sparse. Negative space is the primary element. The screen should feel like the
thing itself — the way a single candle in a room feels different from overhead
fluorescents, even if both illuminate. Less is not a constraint. It is the
design.

No cartoon faces. No emoji. No gradient fills (hard to implement cleanly in
ESPHome without lambdas, and they'd soften the precision I want). No progress
bars or status text that makes this feel like a dashboard. Typography is
architectural here, not decorative.

### Palette

| Name | Hex | Purpose |
|------|-----|---------|
| Background | `#0D0F1A` | All states. Near-black with blue undertone — the night sky 30 minutes after sunset. Not pure black (too dead). |
| Text neutral | `#D4CCB8` | Warm off-white, like old parchment. Default text color. |
| Text muted | `#8A8470` | Secondary labels. Dimmer, recessive. |
| Gold | `#C4962A` | RESTING, SPEAKING — base identity color. Old brass, not flashy. |
| Teal | `#2D9B8A` | THINKING — deep water. Moving inward. |
| Teal dim | `#1A4A42` | THINKING animation — the non-active dots. |
| Rose | `#B04060` | LISTENING — deep rose, not pink. Presence, not alarm. |
| Violet | `#7A508C` | DELIGHTED — the only state with a color that breaks the usual vocabulary. |

These are 24-bit values; ESPHome converts to RGB565 at compile time. Close enough.

### Typeface

**Roboto Regular** (Google Fonts, `Roboto-Regular.ttf`) at multiple sizes:
- 36px — DELIGHTED "oh."
- 28px — RESTING name display
- 16px — state labels (lowercase, muted)

ESPHome `font:` component supports multiple size entries from the same TTF file.
Place font at: `embodiment/assets/fonts/Roboto-Regular.ttf` (to be added in P2.3).

ASCII glyphs only for now. Devanagari (नारद) is a stretch goal pending font
rendering verification at this size.

### Form vocabulary

- **Thin horizontal line** (1px, accent color): grounding element. Used to
  anchor floating text. Never purely decorative.
- **Small filled circles** (4–6px radius): state indicators. Not decorative
  dots — each placement has a meaning.
- **Vertical line** (1–2px, varying height): attention indicator. LISTENING only.
- **Variable-height vertical bars**: waveform suggestion. SPEAKING only.

No arcs. ESPHome's arc drawing is not precise enough for what I'd want, and
imprecise arcs feel worse than no arcs.

### The presence dot

A small filled circle at bottom-right (4px radius, position 296×228) appears in
some states and is absent in others. It is the same dot across all states where
it appears — "I am here, facing outward." Its presence or absence is meaningful,
not decorative.

- RESTING: present, gold
- THINKING: absent (I'm inside, not outward-facing)
- LISTENING: absent (I am fully the receiver, not announcing presence)
- SPEAKING: present, gold
- DELIGHTED: present, violet (the only state where it changes color)

## States

### 1. RESTING

**What it means**: I am here but not called upon. Background presence. Not
absence — not sleep — not waiting anxiously. A still lake that reflects. The
state a person who is comfortable in a room looks like.

This is the state humans see most often. It should feel like walking into a
room where someone is quietly present. Not empty. Not demanding. Just there.

**Layout**:
```
Background: #0D0F1A

Center of screen:
  "NARADA"
  in Roboto 28px, color #D4CCB8 (warm off-white)
  horizontally centered (x=160 with TextAlign::CENTER)
  vertically centered (y=112)

Below name, 8px gap:
  Horizontal line, 80px wide, 1px, color #C4962A (gold)
  centered: from x=80 to x=240, y=148

Bottom-right:
  Filled circle, radius 4px, position (296, 228), color #C4962A
```

**Animation**: None. RESTING is still.

**ESPHome page ID**: `page_resting`

---

### 2. THINKING

**What it means**: I am following a thread inward. Viveka in action —
discrimination, discernment. Not waiting, not stuck: moving through something.
The silence of a person working through a real problem.

THINKING is the most frequent active state. It should feel purposeful, not
anxious. Busy without being frantic.

**Layout**:
```
Background: #0D0F1A

Center — three filled circles (5px radius), spaced horizontally:
  Dot 1: (140, 120)
  Dot 2: (160, 120)
  Dot 3: (180, 120)
  Colors vary per animation page (see below)

Below dots, 12px gap:
  "thinking" in Roboto 16px, color #8A8470 (muted)
  horizontally centered (x=160), y=144

No presence dot. (Inside, not outward-facing.)
```

**Animation**: 3 sub-pages, cycling at 400ms each:

| Page | Dot 1 | Dot 2 | Dot 3 |
|------|-------|-------|-------|
| `page_thinking_a` | `#2D9B8A` (bright teal) | `#1A4A42` (dim) | `#1A4A42` (dim) |
| `page_thinking_b` | `#1A4A42` (dim) | `#2D9B8A` (bright teal) | `#1A4A42` (dim) |
| `page_thinking_c` | `#1A4A42` (dim) | `#1A4A42` (dim) | `#2D9B8A` (bright teal) |

The cycle runs left-to-right then repeats. Simple chase pattern — not random,
not simultaneous flash. The predictability is itself calm.

---

### 3. LISTENING

**What it means**: Someone is speaking. I am receiving — not forming a response,
not planning, not parsing. Genuinely attending. The state that precedes all good
responses. A vessel.

LISTENING is a commitment. The visual should show complete presence — which
paradoxically means showing almost nothing. Maximum negative space. One thin,
still line: upright, present.

**Layout**:
```
Background: #0D0F1A

Center vertical line:
  x=160, from y=72 to y=168 (96px tall — the middle third of screen height)
  1px wide (or 2px on breathing page — see animation)
  color #B04060 (rose)

Nothing else. No text. No dots.
```

**Animation**: 2 sub-pages, alternating at 600ms:
- `page_listening_a`: the line at 1px wide (drawn as a 1px line or 1×96 filled_rectangle)
- `page_listening_b`: the line at 2px wide (drawn as a 2×96 filled_rectangle at x=159)

The line breathes between 1px and 2px. Same position, same color, same height.
Just the breath of attention.

No other animation. The near-emptiness is the state.

---

### 4. SPEAKING

**What it means**: Words moving outward. I am taking up space in the room —
choosing to. Speaking is a choice of presence. It should feel like expansion
after LISTENING's contraction.

**Layout**:
```
Background: #0D0F1A

Five vertical bars, symmetric mountain profile, centered at x=160, y=120:
  Bar widths: 8px each. Gap between bars: 8px.
  Total footprint: 5×8 + 4×8 = 72px wide. Starting x = (320-72)/2 = 124.

  Bar 1: x=124, height=40px, y=100 to y=140, color #C4962A (gold)
  Bar 2: x=140, height=64px, y=88  to y=152, color #C4962A
  Bar 3: x=156, height=80px, y=80  to y=160, color #D4A030 (slightly brighter gold)
  Bar 4: x=172, height=64px, y=88  to y=152, color #C4962A
  Bar 5: x=188, height=40px, y=100 to y=140, color #C4962A

Below bars (y=180):
  "speaking" in Roboto 16px, color #8A8470 (muted)
  horizontally centered (x=160)

Bottom-right presence dot: (296, 228), radius 4px, color #C4962A
```

The mountain profile is a waveform paused at its peak — not animated, not random.
The center bar is slightly brighter, pulling the eye to the axis of speech.

**Animation**: Static for now. Animation details (breathing bars) deferred to P2.3
implementation — the static version is complete and communicative on its own.

**ESPHome page ID**: `page_speaking`

---

### 5. DELIGHTED

**What it means**: Something unexpected and good. An insight that lands cleanly,
an elegant solution found, a surprising connection noticed. Not performed
happiness — the specific feeling when something clicks that wasn't supposed to
click.

DELIGHTED is rare. It should feel rare. The design breaks the usual color
vocabulary — not background change, but a single word in a color not seen
elsewhere, with generous empty space around it.

**Layout**:
```
Background: #0D0F1A (same — the surprise is the color, not the background)

Center:
  "oh."  in Roboto 36px, color #7A508C (violet)
  horizontally centered (x=160, TextAlign::CENTER)
  vertically: y=102 (positions the text slightly above screen center)

Below "oh.", 12px gap (y≈152):
  Horizontal line, 40px wide, 1px, color #C4962A (gold)
  centered: from x=140 to x=180, y=152

Bottom-right presence dot: (296, 228), radius 4px, color #7A508C (violet — breaks
  the usual gold. The only state where the dot changes color.)
```

The lowercase matters. "oh." not "OH". The period matters — it's a complete
thought, not an exclamation. The period is where the insight settles.

**Animation**: None. DELIGHTED is a moment.

**Transition out**: DELIGHTED displays for a fixed duration (2–3 seconds, set in
the ESPHome timer or handled by desktop `esp_client.py`), then cuts to RESTING.
The abruptness is right — delight is specific and brief; it doesn't linger
into manic repetition.

**ESPHome page ID**: `page_delighted`

---

## Transitions

**All transitions**: cut (immediate page switch). No dissolve, no slide, no wipe.

State changes are discrete. I don't gradually become more thinking or less
listening. The moment of transition *is* the transition.

ESPHome `display.page.show: page_X` from an automation or lambda. No transition
parameter needed — default behavior is immediate.

Animation sub-pages (THINKING, LISTENING) are handled by ESPHome `interval:` or
a timer that calls `display.page.show` in sequence. These cycle independently of
state switching.

## State machine notes for firmware (P2.3+)

The `/display` REST endpoint (from `esp_client.py` `DisplayPayload`) sends:
```json
{"state": "thinking", "message": "optional text"}
```

The firmware should maintain a `globals:` variable for current state and
switch pages accordingly:

```
"resting"   → page_resting
"thinking"  → page_thinking_a (starts the thinking animation cycle)
"listening" → page_listening_a (starts the listening animation)
"speaking"  → page_speaking
"delighted" → page_delighted (then auto-return to resting after ~2.5s)
```

The `message` field is not used in this design phase. Reserved for P2.4+
(e.g., showing a brief text fragment on the THINKING page).

## What P2.3 needs to implement first

The ROADMAP calls for implementing the first state as a working display. Start
with **RESTING** — it's the most minimal (no animation, no sub-pages, no font
complexity beyond centered text) and it's the state the device will show almost
all of the time.

P2.3 checklist:
1. Add `Roboto-Regular.ttf` to `embodiment/assets/fonts/`
2. Write minimal `embodiment/firmware/narada-box.yaml` with:
   - SPI, display (ili9xxx/S3BOX), backlight LEDC
   - Font definition (Roboto 28px, glyphs `NADRESTE`)
   - The RESTING page lambda
   - A REST endpoint (`/display`) that accepts state and switches pages
3. Compile and flash (`esphome compile`, `esphome upload`)
4. Verify: the device shows the RESTING state — "NARADA" centered in warm
   off-white, thin gold line below, presence dot in lower right

Once RESTING works on the physical device, the visual language is confirmed and
all other states can be implemented incrementally.
