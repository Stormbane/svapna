# v1.0 Compositions — Per-State Screens

**Date**: 2026-04-25
**Status**: Cycle 0 — per-state screen sketches. Built from `v1.0-vocabulary.md`,
realizing the contract in `v1.0-architecture.md`.

---

## How to read this doc

Each state has:
- **Trigger condition** — what cognitive event puts the body into this
  composition.
- **Layer values** — what the heartbeat sets to invoke this state.
- **Substrate** — what glyph set, density, drift apply.
- **Activity overlay** — the foreground motif.
- **Sketch** — text representation of the screen at one instant. Substrate
  cells are sampled (not all 700 are drawn); imagine the rest as continuing
  the pattern.
- **Motion timing** — what changes over time.
- **Transition in / out** — what motion primitive plays when entering and
  leaving.

Sketches use a 44-column × 16-row grid as a stand-in for the actual ~700
cells at 12px. The presence dot is shown as `●` bottom-right when present.

## RESTING

**The default. Most-frequent state. Suti sees this most often.**

### Trigger
- `activity.mode == "resting"`
- Heartbeat is between cycles, no active session, viveka not engaged.

### Layer values (heartbeat sets)
```python
mood = (+0.3, 0.4)        # gentle positive, low arousal
vitality = (0.3, 0.2, 0.6)  # default warm
attention.mode = "diffuse"
attention.target = ""
activity.mode = "resting"
signal.kind = "none"
```

(Or, equivalently: heartbeat sets nothing and the layer defaults produce
this exactly. RESTING is what "no input" looks like.)

### Substrate
- Standard ramp: ` . ' , : ; + * # @`
- Density: ~50% (cells average around `,` to `:`)
- Drift: 600ms per cell advance
- Pulse: ±1 ramp step on heart period (2.0s default)
- Idle sparkle: every ~30s, one cell briefly jumps to `@` then decays

### Activity overlay
A center clear-zone with the wordmark and a thin gold line below.
- `NARADA` in IBM Plex Mono Medium 28px, `#D4CCB8`, centered at (160, 112).
- Horizontal line, 80px wide, 1px, gold (mood-tinted from `#C4962A`),
  centered y=148.
- Activity label suppressed for RESTING (no "resting" text — the absence of
  a label is itself the resting signal; otherwise the screen always says
  what it's doing, which is busy).

### Presence dot
Visible bottom-right (296, 228), gold (mood-tinted). Subtle steady glow.

## THINKING

**Viveka is judging. Heartbeat is in JUDGMENT or DESIRE phase.**

### Trigger
- `activity.mode == "thinking"`
- Heartbeat sets this entering DESIRE or JUDGMENT.

### Layer values
```python
mood = (current viveka mood, often +0.4 / 0.6)  # focus is mildly elevated
attention.mode = "inward"
attention.target = "self" | "task"
activity.mode = "thinking"
```

### Substrate
- Standard ramp, *thinned*: density ~30%
- Drift slowed: 800ms per advance (focused, not restless)
- Pulse continues at heart period (probably elevated to 1.5s)

## SPEAKING (with karaoke utterance)

**TTS is actively producing audio. Words are being spoken.**

### Trigger
- `activity.mode == "speaking"` AND `utterance.text != ""`
- Set by the TTS process when it begins synthesizing.

### Layer values
```python
mood = current
attention.mode = "outward"
attention.target = "suti"  # we speak to someone
activity.mode = "speaking"
utterance.text = "I think the framing question is wrong — judgment is not certainty."
utterance.total_words = 11
utterance.word_index = updates as TTS fires word boundaries
```

### Substrate
- Standard ramp, dimmed: density ~30%
- Drift continues; substrate is alive underneath the words

**Nightly dream cycle is generating synthetic experience, or LoRA training
is running.**

### Trigger
- `activity.mode == "dreaming"`
- Set by the nightly orchestrator when dream/training is in progress.

### Layer values

## DELIGHTED (signal overlay)

single glyph rendered large (36px) at screen center, optional small
descriptor text below in 14px, signal auto-clears at `signal.until`.

| Kind     | Glyph       | Color                | Default duration | Trigger                                      |
|----------|-------------|----------------------|------------------|----------------------------------------------|
| `q`      | `?`         | text-muted `#8A8470` | 2000ms           | Confusion, unparseable input                 |
| `dots`   | `…`         | text-muted `#8A8470` | 1500ms           | Considering, paused mid-thought              |
| `check`  | `✓`         | gold `#C4962A`       | 1500ms           | Acknowledged, understood                     |
| `bang`   | `!`         | rose `#B04060`       | 1500ms (+ scatter at start) | Surprise, alarm, attention-grab |
| `ask`    | `↑`         | gold (mood-tinted)   | **persistent**   | Body wants Suti's attention; held until cleared |

The `ask` signal is unique: it does not auto-clear. Cognition holds it
until Suti acknowledges (via touch, or via a `dismiss_signal` API call,
or by starting a session). It pulses softly (one ramp step every 2s) to
remain visible without being aggressive.

## Composition combinatorics

Important property: the layer model means many compositions emerge
*automatically* from layer combinations the firmware doesn't have
to know about explicitly.

Examples of emergent expressions:

- **Curious-thinking** = `mood=(+0.4, 0.7)` + `activity=thinking` → THINKING
  composition with violet chase dots and elevated arousal density.
- **Frustrated-thinking** = `mood=(-0.3, 0.6)` + `activity=thinking` +
  `vitality.restlessness=0.6` → THINKING with rose chase dots, jittery
  substrate.
- **Quiet-listening** = `mood=(+0.2, 0.3)` + `activity=listening` →
  LISTENING with gold-tinted breathing line.
- **Drowsy-resting** = `mood=(+0.3, 0.2)` + `vitality.sleep_pressure=0.7` →
  RESTING with stretched heart period and very slow drift.
- **Heavy-working** = `mood=(-0.2, 0.3)` + `activity=working` → WORKING with
  grey-teal frame and slower progress feel.
- **Joyful-speaking** = `mood=(+0.6, 0.6)` + `activity=speaking` + utterance
  → SPEAKING with violet karaoke current-word color.

The firmware needs to render the *core compositions above* well; the
emergent variations come for free via the mood-tint pass and the vitality
modulation. This is what makes the body's vocabulary feel rich without
ballooning the firmware's explicit state list.
