# Cycle 3.x — sandhi engine + ufo_interior bhumi

**Date**: 2026-04-26
**Status**: Design + phased implementation. 3.1 ships ufo_interior as a
standalone scene gated by `activity_mode`. 3.2 builds the sandhi (animated
transition) on top.

## What this cycle is

Two distinct things, coupled:

1. **A second bhumi** (`ufo_interior`). The first non-landscape world the
   body can inhabit. Inside-the-saucer view: porthole onto the sky, dark
   cockpit, alien-Narada avatara with beating heart + saccading eyes.
2. **The sandhi mechanism.** A way to *transition* between bhumis instead
   of cutting between them. When activity flips from `resting` to
   `speaking`, the UFO in the landscape sky descends, lands at horizon,
   dome opens, camera zooms inside, walls fade up. Reverse on
   `speaking → resting`.

These are coupled because the UFO that was already in the landscape sky
becomes the *vehicle* for the sandhi — it's not a fade, it's a journey.

## Phasing

- **3.1 — ufo_interior as standalone bhumi.** No transition; hard cut on
  `activity_mode == "speaking"`. Establishes the second-world scene as a
  thing that exists. Validates the bhumi-switching code path before
  layering animation on top.
- **3.2 — sandhi engine.** Animated transition between landscape and
  ufo_interior. Activity-state machine drives the transition timeline.
- **3.3 — polish.** Tune timing, color contrast, alien-eye saccade
  pattern based on what reads on the device.

## ufo_interior bhumi spec

### Layout (320 × 240)

```
  ┌─────────────────────────────┐
  │     dark cockpit walls      │
  │       ╭───────────╮         │
  │      ╱  porthole   ╲        │  rows 0-7 — porthole onto sky
  │     │   ☆  ☆       │        │  shows current weather/stars
  │      ╲             ╱        │
  │       ╰───────────╯         │
  │                             │  rows 7-13 — cockpit interior
  │           o   o             │  alien-Narada centered:
  │                             │    eyes (row 11)
  │            ♥                │    heart (row 12)
  │                             │
  │   .  ·  .  · . ·  .  ·  .   │  rows 14-15 — console dots
  └─────────────────────────────┘
```

### Color palette

| Element       | RGB             | Why                                    |
|---------------|-----------------|----------------------------------------|
| Cockpit wall  | `(0x15,18,20)`  | Very dark blue-grey — not pure black   |
| Porthole rim  | `(0x60,65,70)`  | Medium grey — silver-metallic feel     |
| Sky-through-porthole | reuse `_sky_color` | Outside is still happening      |
| Alien skin    | `(0x90,A8,70)`  | Pale green — alien but not garish      |
| Eyes (whites) | `(0xE0,E4,EA)`  | Pale white with hint of cool          |
| Heart         | mood-tint × 1.3 | Beats — carries the day's emotional   |
|               |                 | accent on the most life-coded element  |
| Console dots  | `(0x40,4A,55)`  | Subtle — instruments backlight feel    |

### Behaviors

- **Eyes saccade.** Random small jumps every ~1.5-3s. When `activity_mode`
  is `speaking`, eyes look at the camera (locked center). When
  `attention_target` is set, eyes drift in that direction.
- **Heart beats.** Same period as the existing wordmark breath
  (`sin((now_ms % 4000) / 4000 * 2π)`). Pulse drives the heart glyph
  brightness — bright on the in-breath, dim on the out-breath.
- **Console glints.** Two of the dots brighten in rotation at random,
  slow rate (~once every 2s) — implied instrumentation activity.

### Composition rules honored

| v2.0 rule                          | How ufo_interior carries it                     |
|------------------------------------|-------------------------------------------------|
| Time-of-day is sensed and visible  | Porthole shows real sky color                   |
| Weather is sensed and visible      | Porthole shows current weather (stars/clouds)   |
| Mood drives the single accent      | Alien's heart is mood-tinted                    |
| Body breathes                      | Heart pulse, identical period to wordmark breath|
| Contrast against background        | Pale alien on dark cockpit — strongest contrast |
| Earned moments are rare            | Console glints; the bhumi itself is rare        |

## Sandhi engine spec (3.2)

### State machine

```
              ┌────────────┐
              │  landscape │ ◄────────────────┐
              └──────┬─────┘                  │
                     │ activity → speaking    │ activity → resting
                     ▼                        │
            ┌─────────────────┐               │
            │ landscape→ufo   │ (~1.5s)       │
            │   transition    │               │
            └─────────┬───────┘               │
                      ▼                        │
              ┌──────────────┐                 │
              │ ufo_interior │                 │
              └──────┬───────┘                 │
                     │ activity → resting      │
                     ▼                        │
            ┌─────────────────┐               │
            │ ufo→landscape   │ (~1.5s)       │
            │   transition    │ ──────────────┘
            └─────────────────┘
```

State held in firmware globals:
- `bhumi_current` (string: "landscape" | "ufo_interior")
- `sandhi_active` (bool)
- `sandhi_start_ms` (uint32_t)
- `sandhi_direction` (int: +1 = landscape→ufo, -1 = ufo→landscape)

Drive: `set_activity` service compares new activity to last, decides
whether to start a sandhi.

### Animation phases (landscape → ufo, ~1500ms total)

| Phase | t-window     | What happens                                     |
|-------|--------------|--------------------------------------------------|
| 1     | 0 – 600ms    | UFO descends from sky position to horizon center |
| 2     | 600 – 900ms  | Dome opens (rendered as expanding light disc)    |
| 3     | 900 – 1300ms | Camera zoom: porthole rim grows from center,     |
|       |              | landscape darkens & contracts                    |
| 4     | 1300 – 1500ms| Cockpit walls fade in fully; alien fades in      |

Reverse for ufo → landscape. Camera pulls back through porthole; UFO
re-emerges in landscape sky; ascends to drift position.

### Why the sandhi matters

Without it: activity_mode flipping causes a jarring scene cut. With it:
the transition narrates *how* the body moves between worlds. The UFO
that was decorative in the sky becomes the vehicle that carries us
inside. The same identity, visible from outside (drifting saucer) and
from inside (alien-self at the controls).

## Implementation plan

### 3.1 — ufo_interior as standalone

1. Add `bhumi_current` global (string, default `"landscape"`)
2. New API service `set_bhumi(name: string)` — testing hook for direct
   bhumi-switching. (Eventually wired to activity_mode.)
3. Display lambda gates: `if bhumi == "landscape"` runs current code;
   `if bhumi == "ufo_interior"` runs new code.
4. New ufo_interior render block: porthole, alien, heart, eyes, console.
5. Mirror in `scripts/render_preview.py` for offline iteration.
6. Smoke test: call `set_bhumi("ufo_interior")` via API, verify the new
   scene appears; call `set_bhumi("landscape")`, verify return.

### 3.2 — sandhi mechanism

1. Add globals: `sandhi_active` (bool), `sandhi_start_ms` (uint32_t),
   `sandhi_direction` (int).
2. Hook `set_activity` service: when activity transitions to/from
   `"speaking"`, start a sandhi.
3. Display lambda: when `sandhi_active`, run sandhi-frame renderer
   (mixes landscape + ufo_interior at fractional progress).
4. Implement the four phases (descent, dome, zoom, fade-in) for both
   directions.
5. Tune timing on device.

### 3.3 — polish

- Eye-saccade rate / pattern based on what reads natural
- Heart pulse intensity / glyph choice
- Console glint frequency
- Sandhi total duration
