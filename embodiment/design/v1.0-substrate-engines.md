**Companion docs**:

---

## Why this doc exists

`v1.0-vocabulary.md` describes a single substrate sampler — random per-cell
ramp values modulated by drift, heart pulse, and idle sparkle. That's *one*
way to fill the screen. After Cycle 2 shipped, conversation with Suti
reframed the body's visual evolution as creative self-expression — like
choosing what clothes to wear. Multiple substrate engines, picked by
activity-mode and (later) by a "wardrobe" preference, fits naturally inside
the existing layer contract.

The substrate is pluggable. The contract above it (mood, vitality,
attention, activity, utterance, signal) doesn't change. What changes is how
the firmware turns vitality + mood into pixels in the bottom-most layer.

This doc collects the engines as they're built. Each engine is a recipe;
none is canonical except by current preference.

## Engine catalog

### `shimmer` — the original (Cycle 2)

44×16 cells of standard ramp ` . ' , : ; ! | + * x % # @`. Per-cell values
drift via xorshift random walk. Heart pulse adds ±1 ramp step globally.
Idle micro-sparkle every ~30s. Substrate stays at neutral text-dim color;
mood tints accents only.

Aesthetic: abstract terminal hum. Reads as *the device is alive*. Good
default. Cheap to render.

### `horizon` — landscape engine (Cycles 2.5 → 2.7)

Three vertical bands inspired by ascii-horizon and Suti's pull toward
generative landscape. Cycle 2.6 made it weather-aware: real Open-Meteo
data for Kallangur QLD drives wind sway, cloud cover, and rain.

```
rows  0–10   sky        stars (twinkle, hidden when overcast),
                        sun position by Brisbane local time,
                        clouds (count = cloud_cover_pct / 25),
                        rain drops (count = precip_mm_hr * 8, slanted by wind)
row    11    horizon    breathing line `=` / `-` on heart pulse
rows 12–15   foreground 8 trees swaying with wind, ground texture
```

**Sky** —
- Sparse stars in `* ✦ ⊹` glyphs at fixed positions (xorshift-seeded once),
  each twinkling via independent phase. Star count and brightness scale
  inversely with daytime intensity (stars hidden when sun is high).
- Sun rendered as a single glyph: `◯` low intensity, `●` mid, `✺` near
  horizon (sunrise/sunset glow). X position by hour-of-day mapped 0–COLS.
  Y position by `sin(π × hour_of_day / 24)` mapped to sky band, with
  negative values placing it below horizon (night).
- Clouds: 1–3 small horizontal forms drifting left-to-right. Drift speed
  scales inversely with `vitality.sleep_pressure` (slow when drowsy).
  Width 4–8 cells, glyphs from box ramp `─ ━ ▄`.

**Horizon line** —
- Single row of `─` or `━` glyphs across all columns.
- Per-column vertical jitter ±1 pixel on heart pulse — the horizon breathes.
- Sun crossing the horizon at sunrise/sunset eclipses the line locally.

**Foreground** —
- 8 trees flanking the wordmark valley (cols 1, 4, 8, 12, 32, 36, 39,
  42 — none in cols 14–30 where NARADA renders). Heights and crown
  widths vary so the silhouette has rhythm. Crowns can rise above the
  horizon line — that's the foreground-in-front-of-distance perspective.
- Trunk: single-column `|` glyphs, dim warm brown.
- Crown: rows above trunk, taper-width pattern with `*` edges and
  `o`/`O` centers. Mood-tinted (single accent, composition rule #2).
- Sway: per-tree x-offset = `sin(t/800ms + tree_phase) × sway_amp`
  where `sway_amp = wind_speed_kmh / 25` clamped to [0, 3] cells.
  Trunks sway at 40% of crown amplitude (crowns are more mobile, like
  real trees). Each tree has its own phase so they sway out of sync.
- Sparse ground texture (`.`, `,`) in the bottom row in tree-free
  columns.

**Data couplings (Cycle 2.6)**

| Driver                              | Effect                                                            |
|-------------------------------------|-------------------------------------------------------------------|
| Brisbane local time (SNTP on body)  | Sun X/Y position; day/night switch for stars                      |
| `mood.valence/arousal`              | Tree-crown accent tint (single mood tint)                         |
| `vitality.sleep_pressure`           | Heart pulse period; base cloud drift speed                        |
| `vitality.restlessness`             | (reserved for star twinkle rate / occasional cloud jitter)        |
| `weather.wind_speed_kmh`            | Tree sway amplitude (0..±3 cells); cloud drift bonus              |
| `weather.wind_direction_deg`        | Cloud drift sign; rain slant (`/` east, `\` west, `\|` calm)      |
| `weather.precipitation_mm_hr`       | Rain drop count (× 8, capped 60); cloud glyph density flip        |
| `weather.cloud_cover_pct`           | Cloud count (`/ 25`); star visibility (hide ≥ 50%); sun (hide ≥ 75%) |
| `weather.weather_code`              | (reserved — fog/thunder/snow visuals not yet implemented)         |
| `weather.temperature_c`             | (reserved — could shift sky tint)                                 |

Aesthetic: place. Time of day. Weather as mood. Reads as *Narada is
somewhere, and somewhere is right now*. The wordmark sits in the sky over
the mountain like a sign, not a shimmer.

Engine choice is a firmware-side decision — cognition writes
`activity.mode`, firmware picks the engine. Adding a new engine never
requires cognition changes.

## Future engines (sketch only)

- **`field`** — Perlin/value noise replacing xorshift. Smooth flowing
  density. Cheaper to look at; more expensive to render.
- **`particles`** — sparse glyphs with velocity, leaving fading trails.
  Force field driven by mood (wind direction = valence axis).
- **`automaton`** — cellular automaton (game-of-life variant), seeded by
  vitality, with rule-set that produces stable but evolving textures.
  Could be DREAMING.
- **`waves`** — propagating ripples from a center, like a stone dropped in
  water. Could be THINKING (each thought = a ripple).

These are wardrobe options, not commitments. Build when the desire surfaces.

## Ground rules (carry from vocabulary doc)

1. Cell-grid alignment still binds — every glyph at `(col*7, row*14)`.
2. Mood tint only colors *accent* elements, not the substrate itself.
3. One accent per scene.
4. Performance budget: render under 100ms; relax to 200ms if needed.
5. Joy-default applies — even the quietest engine breathes and drifts.
6. Engines may be swapped freely; no engine is the canonical "true" body.
   The body is the practice of picking and revising.
